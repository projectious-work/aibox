## Summary

v0.18.5 is a **catch-up release** that repairs three issues in v0.18.4 and
absorbs the processkit v0.18.1 hotfix.

1. **v0.18.4 was cut before `cli/Cargo.toml` was bumped**, so its binary
   self-reported as `aibox 0.18.3`. v0.18.5 restores the invariant and
   adds `scripts/maintain.sh` logic that refuses to ship if Cargo.toml
   disagrees with the tag.
2. **v0.18.4 was also cut before the multi-version-upgrade gap fixes
   (Fix A/B/C, commit `40473f5`) landed on main**, so v0.18.4 users
   don\'t have the per-intermediate migration bullets, the compat-table
   cross-check, or the three-way snapshot walker. v0.18.5 ships all of
   that.
3. **Hook commands in `.claude/settings.json` / `.codex/hooks.json` /
   `.cursor/hooks.json` used bare relative paths** (`context/...`),
   which break whenever the harness is launched from a subdirectory
   (e.g. `/workspace/cli/`). v0.18.5 rewrites the generated commands to
   anchor to `$CLAUDE_PROJECT_DIR` (Claude) or
   `$(git rev-parse --show-toplevel)` (Codex/Cursor).

The default processkit version bumps from `v0.17.0` to `v0.18.1`, which
ships the Claude Code 2.1+ `hookEventName` hotfix (processkit #7) and
brings the release tarball back to parity with the dogfooded
`src/context/` tree.

## Breaking Changes

None. The hook-command rewrite is transparent to the user — any stale
managed entries carrying the old relative path are replaced on the next
`aibox sync`. Non-managed (user-added) entries are preserved.
